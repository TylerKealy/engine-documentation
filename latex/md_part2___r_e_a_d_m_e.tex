 The {\bfseries{key}} components for your project are\+:


\begin{DoxyEnumerate}
\item The C++ engine that does the heavy lifting and management of game objects and resources
\item A simple to use editor for building games (e.\+g. a GUI-\/based tile editor)
\item A game built in your engine (i.\+e. a class or original game) with 3 \textquotesingle{}scenes\textquotesingle{}.
\end{DoxyEnumerate}\hypertarget{md_part2___r_e_a_d_m_e_autotoc_md19}{}\doxysection{\texorpdfstring{Technical Requirements}{Technical Requirements}}\label{md_part2___r_e_a_d_m_e_autotoc_md19}
The following are the technical requirements of your game engine. There is significant freedom in how you achieve them, but I would like you to apply these techniques in your engine.


\begin{DoxyItemize}
\item \mbox{[} \mbox{]} Implement a resource manager
\item \mbox{[} \mbox{]} Implement a GUI-\/based editor/environment for assisting building a game (e.\+g. A tilemap editor, 2D animation preview tool, etc.)
\begin{DoxyItemize}
\item This tool need not be implemented in C++, but should otherwise generate data that your C++ engine can use.
\end{DoxyItemize}
\item \mbox{[} \mbox{]} Your engine must be data-\/driven
\begin{DoxyItemize}
\item \mbox{[} \mbox{]} Scripts should be loaded for the gameplay logic
\item \mbox{[} \mbox{]} Other configuration files (e.\+g. levels, scenes, etc.) should be loaded at run-\/time.
\end{DoxyItemize}
\item \mbox{[} \mbox{]} Your engine should be component-\/based {\bfseries{or}} use some other logical pattern for organizing game objects (At a minimum you should have a gameobject class).
\item \mbox{[} \mbox{]} Something {\bfseries{extra}} that gives your engine a \textquotesingle{}wow\textquotesingle{} factor to show off to the TA\textquotesingle{}s and instructors. Highlight this in your video (could be engineering, could be a gameplay mechanic that you designed your engine around, the goal is that it is something non-\/trivial)
\end{DoxyItemize}\hypertarget{md_part2___r_e_a_d_m_e_autotoc_md20}{}\doxysection{\texorpdfstring{Gameplan}{Gameplan}}\label{md_part2___r_e_a_d_m_e_autotoc_md20}
